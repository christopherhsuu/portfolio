Batter Archetypes

Clustering MLB hitters into data-driven archetypes using Statcast swing mechanics and contact quality metrics.

Project Overview

This project uses unsupervised machine learning to group MLB hitters into distinct archetypes based on swing mechanics, contact quality, and bat-tracking data. Rather than categorizing players by outcomes alone, the clustering reveals how different hitters generate their results.

The pipeline combines PCA for dimensionality reduction with K-Means clustering to identify archetypes like High-Launch Sluggers or Compact-Contact Hitters. An interactive Streamlit app lets users explore each archetype, view representative players, and compare feature profiles.

Key Technologies

Python • Statcast data • PCA • K-Means • Streamlit • pandas • scikit-learn

Development Process

Step 1: Data Collection & Preparation

Raw Statcast data is extracted from Baseball Savant and processed to create player-level aggregates. Each player's swing-level data is summarized to capture both central tendencies and variation.

Key aggregations:

  • Launch speed, angle (mean, std, median, 95th percentile)
  • Bat speed metrics and hard-hit rate
  • Swing path characteristics (length, attack angle, tilt)
  • Contact positioning relative to the ball

Missing values are imputed or dropped. Outliers are winsorized at the 1st and 99th percentiles. Angular features like attack direction are converted to sine/cosine pairs to handle circular geometry properly.

Step 2: Feature Engineering & Scaling

Features are selected to capture how players swing (mechanics), make contact (quality), and generate outcomes (batted ball types). All features are scaled to prevent any single metric from dominating the clustering.

Feature categories:

  • Contact quality: whiff rate, contact rate, hard-hit percentage
  • Batted ball: launch speed/angle statistics, ground ball/line drive/fly ball rates
  • Swing mechanics: bat speed, swing length, attack angle, path tilt
  • Angular features: attack direction (sin/cos decomposition)

All features are normalized with RobustScaler to reduce the impact of outliers. This ensures the clustering algorithm weights each dimension appropriately.

Step 3: Dimensionality Reduction & Clustering

PCA reduces the feature space to its most important components, making visualization and clustering more effective. K-Means then groups players with similar swing and contact characteristics.

The clustering process:

  1. Apply PCA to reduce to 2-3 principal components
  2. Initialize K-Means with k random centroids
  3. Iteratively assign players to nearest centroid and recalculate centroids
  4. Repeat until clusters stabilize

The result is a set of distinct player archetypes. Each archetype represents a group of hitters who generate results in similar ways, based on their swing mechanics and contact patterns rather than just their outcomes.

Step 4: Evaluation & Visualization

Since clustering is unsupervised, evaluation focuses on interpretability and cluster quality rather than predictive accuracy. Multiple diagnostics ensure the archetypes are meaningful and well-separated.

Evaluation metrics:

  • Silhouette Score: measures how well each player fits within their assigned cluster
  • Davies–Bouldin Index: quantifies cluster separation
  • Centroid heatmaps: show what differentiates each archetype
  • PCA scatter plots: visualize cluster separation in 2D space
  • Representative players: identify prototypical players for each archetype

Step 5: Interactive Deployment

The final pipeline is deployed as a Streamlit app, allowing users to explore archetypes, search for specific players, and visualize cluster characteristics interactively.

App features:

  • Browse all archetypes with centroid visualizations
  • Search for individual players and view their archetype assignment
  • Compare feature profiles across different archetypes
  • View representative players for each cluster

Technical Highlights

  • Circular variables (angles) decomposed into sin/cos pairs to prevent discontinuities
  • Fixed random seeds ensure reproducible clustering across runs
  • PCA and scaler models persisted for consistent transformations on new data
  • Modular design enables easy extension to supervised learning tasks
