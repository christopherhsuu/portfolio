StatForecast

A forecasting tool that uses historical sabermetrics and machine learning to estimate future player performance. This page links to the live app and the code repository.

Launch (Live) Code

Purpose

StatForecast predicts next-season offensive statistics for MLB hitters. The goal is to produce explainable, trajectory-aware forecasts that use multi-year features (lags, rolling means, and deltas) rather than a single-season snapshot.

Quick start

  1. Clone the repository and install dependencies.
  2. Place your hitters CSV (Fangraphs-style) in the repo root or set the path in load_data().
  3. Train the model with train_model() or load the provided joblib artifact, then launch the Streamlit app to view forecasts.

Data preparation (what I did)

  1. Stored season-by-season hitters CSVs with all relevant stats (PA, HR, AVG, OBP, SLG, wOBA, BB%, K%, etc.).
  2. Normalized column names and converted percent strings to floats.
  3. Filtered players by plate appearance threshold (default PA ≥ 200) to reduce noise.
  4. Sorted by Name and Season; validated required columns and handled missing data.

Feature engineering (what I built)

  1. Created lag features (1–3 years) for target stats and covariates.
  2. Computed rolling means (3-year) and shifted to exclude the current season.
  3. Derived delta features, average-delta smoothing, and career-change indicators.
  4. Added age interaction terms to capture age-dependent momentum.
  5. Handled missing engineered features by dropping training rows and zero-filling at prediction time.
