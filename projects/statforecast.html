StatForecast

Forecasting next-season hitting statistics using multi-year trajectories and ensemble machine learning models.

Project Overview

This project predicts next-season offensive statistics for MLB hitters by modeling player trajectories over time. Rather than using only current-season stats, the model incorporates multi-year trends, rolling averages, and year-over-year deltas to capture improvement or decline patterns.

The pipeline uses RandomForest regressors to forecast multiple statistics simultaneously (AVG, SLG, OBP, OPS, wRC+, HR, ISO, wOBA). An interactive Streamlit app lets users explore player histories and generate personalized projections.

Key Technologies

Python • Fangraphs/Baseball-Reference data • RandomForest • Multi-output regression • Streamlit • pandas • scikit-learn

Development Process

Step 1: Data Collection & Cleaning

Season-by-season hitter statistics are collected from Fangraphs or Baseball-Reference. Each row represents one player-season with standard offensive metrics.

Preprocessing steps:

  • Normalize column names and data formats
  • Convert percentage strings to numeric ratios
  • Filter players with fewer than 200 plate appearances
  • Sort by player name and season
  • Validate required columns are present

Step 2: Trajectory-Aware Feature Engineering

The key innovation is creating features that capture player trajectories over time, not just current performance. This helps the model distinguish between rising stars and declining veterans.

Engineered feature types:

  • Lag features: 1-3 year historical values for each statistic
  • Rolling means: 3-year averages to smooth variance
  • Delta features: year-over-year changes
  • Career momentum: 3-year trend indicators
  • Age interactions: age × performance deltas to model aging curves

These features embed sabermetric intuition: a 24-year-old improving slugger carries a different predictive signal than a 36-year-old in decline, even with similar current stats.

Step 3: Multi-Output Model Training

A RandomForest regressor predicts multiple offensive statistics simultaneously. This multi-output approach captures correlations between statistics (e.g., players with high OBP tend to have high OPS).

Training process:

  1. Generate all trajectory features for each player-season
  2. Split into train/test sets (85/15)
  3. Train MultiOutputRegressor with RandomForest (200 trees)
  4. Evaluate using RMSE for each predicted statistic
  5. Save model and feature metadata for deployment

The model predicts AVG, SLG, OBP, OPS, wRC+, HR, ISO, and wOBA simultaneously, learning the relationships between these metrics.

Step 4: Interactive Deployment

The trained model is deployed as a Streamlit app where users can search for players, view their historical performance, and generate next-season projections.

App features:

  • Upload custom player datasets or use default data
  • Search for players via interactive dropdown
  • View multi-year performance history
  • Generate next-season forecasts for all tracked statistics
  • Visualize projections with historical context

Technical Highlights

  • Multi-year trajectory features capture aging curves and performance trends
  • Multi-output regression learns correlations between different statistics
  • Feature metadata saved with model ensures consistency at inference time
  • Lag features and rolling means prevent data leakage while using historical context
  • Age interactions model the different predictive signals of young vs. veteran players
